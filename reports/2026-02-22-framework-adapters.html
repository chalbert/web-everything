Key Takeaways

  • Strongest areas: Routing (8/10 covered), Component Model (5/6), DI (3/4), Accessibility (3/4). These are at or above framework parity.
  • Missing Adapters (7): Signals, function components, two-way binding, event modifiers, nested routes, suspense boundaries, text interpolation — all achievable via syntax transforms or thin adapter layers without new core primitives.
  • Biggest gaps: SSR/hydration (4 missing), forms (2 missing), i18n (2 missing), animations (2 missing). These require new feature work, not just adapters.
  • Unique to Web Everything: Invoker Commands integration (no framework has this), Intent protocol (abstract UX contracts), provider-scoped typed events, comment-based virtual elements.
  • Recommended adapter priorities: (1) Two-way binding (bind:attr), (2) Function component factory, (3) Event modifiers (.stop, .prevent, .once), (4) Signals adapter over SimpleStore keys.
  • Recommended feature priorities: (1) Effect/watcher system, (2) Form state management, (3) i18n, (4) SSR pipeline, (5) Animation library.
