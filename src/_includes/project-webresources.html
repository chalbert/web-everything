Mission

Web Resources aims to decouple the specification of a data request from its execution. It provides a standard interoperability layer for Middleware (Auth, Logging) and Caching that works across any transport (REST, GraphQL, WebSocket).

Ecosystem Analysis

We analyzed the "Big 4" paradigms in the current ecosystem to identify gaps and interoperability opportunities.

Conclusion: Most libraries couple the Transport (HTTP vs GQL) with the Policy (Cache/Auth). Web Resources standardizes the Policy layer so it can be injected into any Transport.
