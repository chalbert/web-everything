When a discussion **overturns or reframes your approach on the merits** — a principle that changes *how
you'd decide the next case*, not just this one — **propose distilling it to a `memory/` entry in the
moment the reframe lands**. Do not wait for `/closing-session`, and do not wait to be asked.

**Why:** writing the specific decision into its `backlog/` item is **not** durable capture of the
*principle*. A lesson that lives only inside the one item that surfaced it reads as "captured" (the file
exists, the gate is green) but is **invisible to a future session reasoning about a different subject** —
so reusable lessons leak precisely when they look saved. The strongest trigger is the user correcting your
framing on principle (e.g. #1377: "the authoring source of truth is the standard form, impls are removable
adapters" — it reached memory only because the user asked; it should have been offered when the reframe
landed).

**How to apply:** at the moment your framing is overturned, ask — "would a future session facing a
*different* instance of this benefit from what we just learned, and is that lesson anywhere it would
actually look?" If no, propose the concrete memory (name + one-line description) right then. This is also
hardened as a `/closing-session` audit bullet ("Generalizable principles & reframes") as a backstop, but
the gate is the *last* line of defense — catch it live. Related:
[[feedback_authoring_sot_is_the_standard_form]], [[feedback_ratified_decisions_are_reversible_stay_agile]].
