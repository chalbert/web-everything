Invoke the `batch-backlog-items` skill to work several small, agent-ready backlog items in a row in one session — claim → work → close-out each, validating at every seam and stopping on a solid condition (a red gate, the count cap, no eligible item left, a surfaced design fork, or a context seam). Present an ordered batch plan, label the session once, and — after I say go — run the whole batch, reporting a compact ledger. A bare number sets the cap (default 3); a `NNN` or `NNN-slug` seeds the first item of the chain.
