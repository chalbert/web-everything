Conformance Auto-Fix Playground

A live surface for backlog #298 — the interactive front end on the conformance auto-fix loop (#095 engine, #196 model fixer, #293 cost-bound + review hook). It is the sibling of the Code Upgrader, one domain over: the upgrader lifts legacy code onto the standard; this drives a failing conformance suite back to green. Each card runs the whole loop: failure → proposed patch → diff → verify gate → accept / revert.

The verify gate is the moat. A proposed patch is applied, the suite is re-run, and the patch is kept only if its target failure cleared and it introduced no new error — else it is reverted automatically. So a plausible-but-wrong fix is never shipped: trustworthiness is structural, not promised. AI is a swappable provider. A fixer (failure → patch) drops into the same registry whether it is the deterministic reference fixer or a bring-your-own-key model fixer; the model cards below are driven by a scripted client (no key in the browser), through the exact seam a real Anthropic client would use, gated by the same verify pass and the same model-fix cost bound.
