DOM-less DI Host — display: contents

Confirms the layout-neutral provider-element pattern from the DOM-Less Composition research: a <di-scope> element that joins the injector tree to scope dependency injection, but renders display: contents so it adds no layout box — a flex/grid ancestor treats its children as its own.

The trick: display: contents keeps the element in the DOM tree (the injector chain walks DOM ancestors, so a descendant still resolves the provider declared on the scope) while removing its box from layout. DOM-present, layout-absent. The control check proves it is load-bearing: a plain <div> wrapper breaks the same flex row.
