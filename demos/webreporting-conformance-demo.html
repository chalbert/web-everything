Web Reporting Conformance Demo

The runnable proof of the Web Reporting report-model protocol end-to-end: three unrelated producer formats — SARIF, JUnit, and an Istanbul coverage summary — each ingest into one normalized Report pivot, then render through the same v1 renderers (findings table, coverage matrix).

The core claim is the lossy-normalization-hub: the Report schema is the pivot the project never ships as a format — every tool's output is ingested into it, so a single set of renderers displays them all (no bespoke viewer per producer). The export adapters close the loop (an export-then-ingest round-trips on the base vocabulary). The section below asserts each invariant against the live impl.
