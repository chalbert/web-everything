JSX Directive Sugar Playground

The prettier <For each> / <Show when> / <Resource from> components are optional sugar for the canonical <template is="…"> directives — an alternative spelling of the same directive, mapped through one registry. Each card authors the sugar, renders it live (the runtime components build the same DOM as the canonical form), and shows desugar() lowering it to <template is>. See the HTML↔JSX feature mapping (rows 7–8).

This is a spelling layer (Axis-1), not a rendering-strategy change: <For> builds an inert <template is="for-each"> exactly as the canonical spelling does; actually expanding the loop is the parked rendering-strategy axis (Axis-2).
