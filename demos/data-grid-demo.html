Data Grid Playground

The runnable home for the Data Grid block — the interactive twin of the read-only Data Table block. Same tabular data, a different WAI-ARIA APG pattern: role="grid" with cell-level keyboard navigation and a roving tabindex, the concrete consumer of the Focus Delegation intent (strategy roving, orientation both). Each card pairs a key sequence with the cell it lands on, renders the grid there, and runs the same audit the CI conformance suite asserts. The last card is live: click in and arrow around the cells with real DOM focus.

A green badge means the rendered grid satisfies the verified contract: a native <table role="grid"> of role="row" / role="columnheader" / role="gridcell" cells (WAI-ARIA APG Data Grid), with exactly one cell at tabindex="0" — the active cell — and every other at -1 (the roving tabindex). The arrow keys move one cell (clamping at the edges), Home/End jump to the row ends, Ctrl+Home/Ctrl+End to the grid corners, and PageUp/PageDown by a page. The renderer here is a deterministic reference; concrete strategies (virtualization, editable cells, server paging) live in Frontier UI.
