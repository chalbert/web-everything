Loader → Background Task Handoff Playground

The producer half of the Background Task handoff — a WE standard: a real Resource Loader running backgroundLoad() escalates a slow load by dispatching a bubbling background-task-register event that a receiver adopts off-view. The receiving end here is a minimal reference receiver (not Frontier UI's <background-tasks> surface, which is impl and demoed FUI-side) — so each card proves an invariant of the escalation + handoff contract, the cross-boundary seam, rather than the surface's own behavior.

Escalation:async trigger. A load escalates exactly when it crosses the Loader's debounce threshold into loading — the moment the Loader already decided the work was slow enough to surface a pending UI. Fast loads that resolve first never enter loading, so they never register. The threshold is shortened here for a snappy demo. Scenarios and the controllable-promise lever come from the same shared fixture module the unit suite imports, so the badges and CI can't drift. This wires #152 (decoupled per #812).
