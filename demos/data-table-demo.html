Data Table Playground

The runnable home for the Data Table block — the blocks-layer realization of the Collection Operations intent's filter, sort, and group stages (the page stage is the companion Pagination block — compose, never merge). Each card pairs a row set with a contract, renders the table through the reference renderer, and runs the same APG / Intl.Collator / SQL-aggregate audit the CI conformance suite asserts. The last card is live: click a header to cycle its sort, toggle the filter, and watch the aria-live region announce what a screen reader would hear.

A green badge means the rendered table satisfies the verified contract: a native <table> with <th scope="col"> headers, the active sort column projecting aria-sort (WAI-ARIA APG Sortable Table) with sortable-but-unsorted headers at none, text order following Intl.Collator (numeric / sensitivity), empties placed by SQL NULLS FIRST/LAST, filters combined by filterMatch (all / any), and per-group groupSummary aggregates named after SQL. The renderer here is a deterministic reference; concrete strategies live in Frontier UI.
