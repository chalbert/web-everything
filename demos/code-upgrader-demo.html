Code Upgrader Playground

A live surface for backlog #094 — an AI upgrader that takes existing/legacy code and lifts it onto the standard. It is the inverse of Module-as-a-Service: MaaS serves one authored component in many forms; the upgrader takes a legacy component and produces the authored <component> in the first place. Each card runs the shared engine: legacy source → neutral structure (IR) → generated <component> → verify gate → live element.

AI is a swappable provider. The analyzer (code → IR) is registry-backed — the same inject-a-provider shape as the MaaS compiler seam. A deterministic reference provider (no model key) handles one source path — vanilla web components; a bring-your-own-key model provider (#188) drops into the same registry for the messier input the heuristic rejects, with no engine change. Below, the model section is driven by a scripted client (no key in the browser); a real run swaps in the thin Anthropic client. The verify gate is the moat: output is only offered when it re-parses, round-trips to the analyzed structure without drift, and references only real intents — generic AI emits plausible code, this emits checked code. Un-verified output is shown for inspection but never presented as trusted.
