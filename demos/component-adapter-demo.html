Component Adapter Playground

A live test surface for the <component> standard and its intended implementation, the Declarative Component adapter. Each card shows the authored declarative source, the generated class the build-time AST transform would emit, and a live element registered from that definition — with a badge checking the rendered tree equals the authored template. Edit the sandbox below to test your own.

This runs the runtime twin of the build-time transform so it can be exercised in a browser (tier-1: static template + slots + shadow). Reactive bindings and the scope attribute are tier-2 and compose with other standards later — see the open questions.

Unlike the JSX Adapter Playground — which verifies two spellings render to the same HTML (a two-way mirror, source ⇄ source) — this playground verifies a one-way lowering: that the generated class renders the authored template (source → output). The badge here means "the lowering is faithful," not "the formats round-trip."
