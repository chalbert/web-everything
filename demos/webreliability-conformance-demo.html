Web Reliability Conformance Demo

The runnable proof of the CustomRecoveryHandler contract (#1051): a handler inspects an error + its RecoveryContext and either declines (null — the next handler is tried) or owns the recovery with a RecoveryResult (retry / queued / fallback / abort).

The contract is type-only (reliability/contract.ts); the HTTP-retry / circuit-breaker / offline-queue handlers and the customRecovery registry are impl in Frontier UI. So this demo supplies its own in-demo offline-retry handler to prove the contract is realizable. The section below asserts each contract invariant (decline, retry-with-backoff, offline queueing, attempt-cap abort) live.
