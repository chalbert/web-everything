Module-as-a-Service Consumer

The native-only seam of backlog #081, phase 2a. This page is a consumer with no build step: it does a native import('/_maas/user-card.js?form=wc-class') and the returned ES module registers the element on evaluation. The server applied the adapter transform; nothing is bundled or transpiled here. Compare with the Module-as-a-Service Playground, which shows the forms in-process.

v1 serves the wc-class form (self-contained ESM). The jsx form needs the delegated transpile step (phase 2b) before it is import-able; declarative/html are previewable but not JS modules. Import-map / bare-specifier resolution kicks in only once a served module imports another — v1 components are self-contained.
