# Should a solo session (non-/workflow) also run in an isolated worktree lane?

Prepared decision, grounded in a prior-art survey published as [/research/solo-session-lane-isolation/](/research/solo-session-lane-isolation/). The prep corrects the card's premise twice: the isolation primitive is a **lane clone**, not a git worktree (worktree-add is guard-blocked; #1996 ruled "isolation = a clone"), and the go is **already ratified statute** ([we:docs/agent/platform-decisions.md#pr-flow-rollout-mechanism](docs/agent/platform-decisions.md); #1998 names serial-on-main "the interim"). So adoption is **accepted on merit** (gate dissolved — merit conceded to statute, only mechanical capability edges left), and the ratify turn carries three genuine calls — claim locus, confirm-or-amend scope, before-state soundness — plus a batch-confirm of the entry/landing mechanics.

**Which layer:** this is the **agent-workflow / session-tooling layer** — nothing here crosses the WE↔FUI standard boundary, defines a contract a consumer observes, or touches any registry. No Block/Intent/Protocol/Capability classification applies; the ruling codifies as session house rules (an extension of the #1996/#1998 anchors), not as standard vocabulary.

## Adoption — accepted on merit; executes statute via mechanical trigger edges (dissolved gate)

**Why there is no go/no-go to ratify here (dissolved per the merit-conceded rule):** the direction is not open — [we:docs/agent/platform-decisions.md#pr-flow-rollout-mechanism](docs/agent/platform-decisions.md) (#1996, ratified) rules "every **automated** writing session (agent / `/workflow`) works in a clone; `main` is convergence-only for automation", and the #1998 ladder explicitly parks today's serial interactive practice as "the interim" ("the serial `/batch` still commits on the current branch per Rule 104 **as the interim**"). That "(agent / `/workflow`)" scope and its rationale (an ad-hoc agent `main` edit cannot be proven disjoint from a live lane) cover interactive solo agent sessions verbatim — the human-writes-`main` carve-out does not rescue them, because the writer in an agent session is the agent. The prep first framed the residual as a phased go/no verdict; the fresh-context screen flagged that framing **(prio)**: with merit conceded to statute, every phase boundary is a *mechanical capability edge* — the claimed warrant unknown ("the lane verify loop has never run") is exactly what the trigger probe encodes, a trigger edge, not a judgment for a ratify turn. So adoption is recorded as **accepted-on-merit / executes statute**, the human turn on this element compresses to a **batch-confirm of the entry/landing mechanics below**, and the genuine ratify content lives in Forks 1–3.

**Prior-art delta:** hosted agent harnesses (Devin-class, Copilot Workspace, Codex-style tasks) offer *only* the isolated-workspace-converge-via-remote model; jj/Sapling make N-working-copies the first-class posture; the parallel-agent community norm is worktrees + per-instance ports (code isolation needs runtime isolation added — the caveat WE ratified as per-lane port bands). The local delta: WE's primitive is the clone (branch guard), and WE's interim practice is the anomaly the statute already names. Full survey: [/research/solo-session-lane-isolation/](/research/solo-session-lane-isolation/).

**The three trigger edges (mechanical — the flip happens when the capability lands, no human weighing at flip time):**

- **Edge 0 — effective now:** a solo *code-only writing* session (a lone `/next` build, a one-item resolve — WE-only, non-visual surfaces, verification = tests/gate) defaults into a **lane clone** from the existing pool ([we:scripts/lane-pool.mjs](scripts/lane-pool.mjs)), works and commits there on the lane's own `main`, pushes a durable `lane/*` ref, and lands via the merged-tree gate.
- **Edge 1 — flips on the first green lane dev-pair probe:** sessions touching **visual surfaces** (`*.njk`/`*.css`/template-rendered UI) *and* **interactive discussion** sessions (a `/prepare`, a decision turn — where the human reads the rendering item body live) stay on the shared primary (the #1996 Gate-5 fallback posture) until **a lane clone boots its WE dev pair on its band ports (requires spin-off 0, the missing env-load link) and a CLI Playwright before/after (lane-local before, Fork 3) reproduces a real UI diff — bounded by the #1895 transparent-`.fui-card` regression class (#1996's Gate-5 milestone).** The same boot capability restores the human's live-board loop for discussion sessions (the lane-band `/backlog/` board), so both classes flip together on that first green probe.
- **Edge 2 — flips when the entry/closeout helper (spin-off 1) lands:** **cross-repo** solo lanes — lane choreography hand-run is where the documented failure modes live (stale-origin false-drops, stranded `active` claims; [we:.claude/agent-memory/workflow-crossrepo-lanes-falsedrop.md](.claude/agent-memory/workflow-crossrepo-lanes-falsedrop.md)), and the "sync impl origins first" pre-flight is unenforced without it.

Merit record (context for the concession — why the statute is right for the solo case, not a fresh case to weigh): (1) the collision class is real and already destroyed work — `batch-2026-06-29e` clobbered a concurrent `/prepare`'s in-progress files on the shared tree; the #1985 forced invariant removed the destructive *reaction*, isolation removes the *collision* — the agent's writes stop interleaving with the user's live tree, converting silent same-file last-write-wins into loud, mergeable divergence; (2) the user has voiced the direction ("even a solo /next should run in an isolated lane" — the memory that filed this card); (3) the machinery is proven at scale (2026-07-01 run: 16/18 landed, cross-repo couples green, 0 stranded).

**Two recorded costs the lane introduces (weighed, accepted with mitigations — not hidden):** (i) *the lane is blind to the user's uncommitted work by construction* (it resets to origin/main + the claim), so a lane can land changes that semantically conflict with the human's in-flight edits — textual conflicts surface at the landing merge, but semantic ones reach the human at their next commit; mitigation: the solo closeout reports the landed file set so the human can diff it against their in-flight surfaces. (ii) *a solo lane has no integrator layer*: `/workflow`'s central integrator re-verifies lane judgments, and the run history shows lane agents "over-drop, recurringly" on stale-tree misjudgments — the re-verify-before-dropping rule and the abandon-→-`release` closeout rule are therefore **specified into the helper (spin-off 1)**, not left as prose discipline.

Entry/landing sequence (**the batch-confirm target** — the one human glance this element asks for). The claim locus and its publication rule are ruled in Fork 1:

```bash
# on the shared primary — claim locally (visible to local tooling + the board)
node scripts/backlog.mjs claim <NNN>
git add backlog/<NNN>-*.md .claude/skills/batch-backlog-items/claims.json && git commit -m "claim(#<NNN>)"
# publish ONLY origin/main + the claim splice (helper replays the 2-file splice onto origin/main;
# interim fallback: `git push origin main` IS allowed but only after an ahead-check — if the primary
# carries the human's unpushed commits, ask before publishing them, never silently)

# enter the lane (refresh sees the published claim)
node scripts/lane-pool.mjs provision --count=1
cd "$(node scripts/lane-pool.mjs path --lane=1)"

# … work; commit on the lane's own main (Rule 104 composes — the lane has its own HEAD) …

git push origin HEAD:refs/heads/lane/solo-<slug>   # durable ref first (#1995 retry semantics)
git fetch origin && git merge origin/main          # assemble the merged tree (contains later claims too)
npm run check:standards                            # authoritative gate ON the merged tree (#1937 anchor)
git push origin main                               # land (convergence)
# abandonment instead of landing → on the primary: node scripts/backlog.mjs release <NNN> (never leave it active)
# on the primary afterwards: git pull (claims published from origin/main-based splices keep this clean)
```

## Fork 1 — claim & status locus

**Fork-existence:** genuine either/or (reclassified by the prep skeptic from "forced invariant"): two coherent substrates for claim visibility exist and a default must pick one. (a) **primary-locus** — claim-class transitions (`claim`, `release`, `settle`) + the claims ledger (`we:.claude/skills/batch-backlog-items/claims.json`) execute on the shared primary checkout and publish to origin immediately; lanes never claim. (b) **remote-locus** — claim anywhere (including a lane), commit, push immediately; origin's ref-transaction lock serializes racing claimers and the loser fails loud on the already-flipped `open → active` transition. (b) is *coherent* post never-push-removal (#1985 Rung 2's "writers collaborate through the remote", taken literally) — not broken, so this is a real fork, not a ratify.

**Recommended default: (a) primary-locus with immediate publish.** The decisive axis is the **read substrate**: every consumer of claim state today reads the *local shared tree* — the `open → active` guard in [we:scripts/backlog.mjs](scripts/backlog.mjs):133-137, `check:readiness --select`, the human's live `/backlog/` board — so a claim made in a lane is invisible to all of them until a fetch nobody has a reason to run; (a) keeps writes where the reads are, and the immediate publish extends visibility to every other clone's refresh. The orchestrator already practices (a) as #1933 choice 2 ("central pre-claim; lanes never claim" — [we:.claude/skills/batch-backlog-items/parallel-execute.workflow.js](.claude/skills/batch-backlog-items/parallel-execute.workflow.js):554-580). *(b) rejected as the default on current-substrate correctness, not on principle*: it presumes fetch-before-claim discipline and a remote-read tooling layer that doesn't exist; if claim tooling ever moves to remote-read, (b) reopens as the natural end-state. **Publication rule (part of this ruling, re-homed here from the dissolved gate):** the claim publishes as **`origin/main` + the claim splice only — never the primary's whole HEAD**, which may carry the human's committed-but-unpushed work (the impl repos ran 66/317 commits ahead under the old policy; silently publishing that is not the session's call); the interim fallback `git push origin main` is allowed only after an **ahead-check** that asks the human before publishing their unpushed commits. **The `resolve` splice rides the lane's landing commit** — it is file content in the lane's commit stream, and its authority doesn't depend on unshared state (the integrator pattern: central pre-claim, lane carries the resolve). Corollary: an abandoned lane session strands its published `active` claim — the abandon-→-`release` closeout is specified into the helper (spin-off 1), mirroring `/workflow` carry handling.

Skeptic: SURVIVES-WITH-AMENDMENT (both passes) → the classification attack landed: "lane-local claim is *broken*" held only relative to the local-read claim substrate — with never-push removed, a remote-locus claim is coherent. Absorbed by **declassifying the forced invariant into this genuine fork** and ruling for (a) on current-substrate grounds, with (b) recorded as the alternative a future remote-read substrate would reopen. The resolve-rides-the-lane refinement stands.
Screen: clear — the locus of a claim is observable cross-session behaviour (who can double-claim, what the board shows); the (a)/(b) difference is correctness-under-current-consumers, a merit axis; ruling on today's substrate while naming the reopen condition is a scoped ruling, not an impl mandate.

## Fork 2 — steady-state scope: confirm or amend #1996's "every" clause

**Fork-existence:** genuine either/or, framed honestly as **confirm-or-amend of ratified statute**: #1985 explicitly deferred "clone scope (every session vs substantive-work-only)" to #1996, and #1996 ratified "**every** automated writing session works in a clone". So (a) *uniform* **confirms** the ratified clause for the solo case; (b) *content-session carve-out* (sessions touching only `backlog/`/`reports/`/research files stay on the primary permanently) would **amend** it. Both are coherent — (b) is not broken (content files are one-file-per-item by design, the #1145 granularity move) — and a default posture is singular, so the branches cannot coexist.

- **(a) Confirm — uniform in steady state, every solo writing session lanes, including content-only `/prepare`.** Merits: precedent-consistency (the clause is ratified; a permanent carve-out re-opens the ad-hoc-edit race its rationale closed); correctness under drift (a "content" session that then codifies a doc, edits a script, or regenerates a derived artifact silently crosses the line mid-session — a uniform rule cannot be misclassified); and the incident class points here — the files destroyed in `batch-2026-06-29e` *were* content files (an item body, a research topic, a report), so exempting content sessions exempts the proven collision surface. **Phase-1 posture (explicit, from the skeptic):** until Edge 1 (the Adoption section's dev-pair trigger) fires, interactive/content sessions **stay on the primary** — their load-bearing observability mitigation (the human reads the in-progress body on the lane-band board) *cannot boot today* (the env-load link is unbuilt), and phase-1 uniform laning would cost every `/prepare` discussion its hot-reloading item body with nothing in its place. So phase 1 *behaves* like (b) by sequencing; what is ratified as the steady state is (a), flipping on capability, not on a later re-decision.
- (b) Amend — a permanent content-session carve-out. Merits: preserves the human's live-observation loop on the surface they actually watch during a `/prepare`; the shared-file class is designed conflict-free, so the collision the lane removes is rarest exactly here. *Rejected as the steady state*: the misclassification hazard is structural (session scope is not knowable at entry), the proven incident sits on this very file class, and amending a freshly-ratified clause needs a stronger warrant than an observability gap the trigger closes anyway.

**Recommended default: (a) confirm — uniform in steady state; interactive/content sessions sequenced behind the dev-pair trigger (phase 1 = primary).**

Skeptic: SURVIVES-WITH-AMENDMENT (both passes) → two attacks landed. (1) *Framing:* the fork re-litigated a ratified clause as a virgin either/or — rewritten as confirm-or-amend, which is what the decider actually rules. (2) *Merit (vaporware mitigation, sharpened by the second pass):* option (a)'s observability answer ("boot the lane's own board") does not exist in phase 1 — no lane can boot its pair until the env-load link is built — so uniform laning of interactive/content sessions was explicitly resequenced behind the trigger, and the phase-1 posture is now stated rather than implied. The steady-state default (a) stands; the prioritization probe ("(b) = defer laning where it's annoying") was rejected both ways — (b)'s live-observability is a genuine UX axis, (a) wins on correctness/precedent axes.
Screen: clear — which checkout a session's edits appear in and what the human's board shows are observable behaviours; with wiring free on both sides a merit difference remains (misclassification-correctness + precedent-consistency vs live-observability); the phase-1 gate is capability-conditional, not calendar deferral.

## Fork 3 — before-state soundness for the lane UI check (ratify)

**Fork-existence:** the *wiring* question this fork originally carried ("which server does a lane session verify against?") is **statute, not a fork** — #1996's fifth ratified call already rules that a visual-touching lane auto-merges only on a headless Playwright render check "on its booted WE+FUI cross-origin pair" (moved to *Settled by precedent*). The prep skeptic also corrected the "forced" over-claim: a *land-then-verify-with-rollback* wiring (before = `:3000` pre-pull, land, pull, after = `:3000` post-pull) is coherent zero-tooling — it is **statute-barred** (the #1996 gate places the render check *pre-merge*), not broken. What remains genuinely un-ruled — and is ratified here — is the **soundness rule for the *before* state** of the UI 3-musts check in a lane session: (a) lane-local by default vs (b) the primary's `:3000` by default. A default must pick one; (b) is coherent only under a precondition, which is the crux.

**Ratify: (a) the *before* is lane-local by default** — snapshot the lane band on the pre-edit tree (the lane's origin/main reset point) before applying the session's edits; **the primary's `:3000` may serve as the before only as a provably-clean shortcut** (the touched surfaces verifiably unmodified on the primary). The reason (b) loses as a default is #1985's own text: "the shared primary checkout's dirty state is the **normal baseline**" — uniform laning exists precisely so the human keeps working on the primary, so the `:3000`-clean precondition fails *structurally* in the steady state this decision creates; the shortcut is the exception, not the rule. The user's `:3000` is never touched or restarted (composing with the standing dev-server rule). **Wiring status (corrected by the skeptic):** the port plumbing is ratified + *generated* but has a dead link — [we:scripts/lane-pool.mjs](scripts/lane-pool.mjs):101-124 writes each lane's `.env.local`; [we:vite.config.mts](vite.config.mts):105-123 reads `WE_VITE_PORT`/`WE_ELEVENTY_PORT` from `process.env` (`strictPort: true`); [we:package.json](package.json) dev scripts expand the same shell vars — **but nothing loads `.env.local` into the shell/process environment** (repo-wide, its only consumer is its writer plus a vite.config comment), so a lane `npm run dev` today boots on 3000/8080 and `strictPort` collides loudly with the user's live server. The env-load link is spin-off 0 and a precondition of Adoption Edge 1. Tooling note: `/review-design` + the Playwright checks need a base-URL/port parameter (spin-off 2).

Skeptic: SURVIVES-WITH-AMENDMENT (both passes) → three corrections landed and were absorbed: (1) the fork's original subject ("verify against the lane's own pair") was **already #1996 statute** — the fork was narrowed to the genuinely un-ruled before-soundness rule and the wiring moved to *Settled by precedent*; (2) "forced invariant" was over-claimed — land-then-verify is coherent but statute-barred, now said explicitly; (3) "already live end-to-end" was factually false — the `.env.local` → shell → vite chain has a verified dead link, rewritten to *ratified + generated, env-load link missing* and named as spin-off 0; and the lane-local before was promoted into the ratify sentence (per #1985's "dirty primary is the normal baseline"), demoting `:3000` to the provably-clean shortcut.
Screen: clear — which tree the *before* observes is the contract (a correctness merit: an unsound before conflates the human's live edits with the lane's); port values and the env-load mechanism are plumbing, correctly left as build items; no timing content in the rule.

## Settled by precedent (not decisions)

- **Worktree vs clone vs fresh clone (the card's title premise):** settled — the branch guard denies `git worktree add`/`checkout -b` in the shared checkout (proven live, first parallel run), and #1996 codified "isolation = a clone"; the pool is built and proven ([we:scripts/lane-pool.mjs](scripts/lane-pool.mjs)). The industry worktree norm was surveyed and the deviation is repo-local + already ruled; the harness's own `isolation:"worktree"` stays fine for throwaway read-mostly sub-agents. Re-opening the primitive would rule on an impl detail below the session-observable contract ("isolated checkout + `lane/*` convergence").
- **Verify-against-the-lane's-own-booted-pair (the wiring):** settled by [we:docs/agent/platform-decisions.md#pr-flow-rollout-mechanism](docs/agent/platform-decisions.md)'s fifth ratified call (visual lanes land only on a headless render check against their own booted pair); Fork 3 rules only the before-state soundness residue.
- **Landing procedure:** settled by composition, not a fork — durable `lane/*` push first, then merge `origin/main`, authoritative gate **on the merged tree** ([we:docs/agent/platform-decisions.md#gate-on-merged-tree-lane-fast-fail](docs/agent/platform-decisions.md)), then land; on red, the `lane/*` ref preserves the work. The two candidate shapes ("push a ref and stop" vs "self-integrate") compose as stages of one procedure.
- **Per-lane dev-server ports:** ratified by #1996 (deterministic band offset, `strictPort`) and *generated* by #1997 — the env-load link is the one missing build (spin-off 0), not a decision.
- **Cross-repo caveat (operational, carried from the runs):** before a cross-repo solo lane, sync each impl repo's `origin/main` first, or the lane resets to a stale tree and false-drops ([we:.claude/agent-memory/workflow-crossrepo-lanes-falsedrop.md](.claude/agent-memory/workflow-crossrepo-lanes-falsedrop.md)) — enforced by the helper Adoption Edge 2 gates cross-repo lanes on.
