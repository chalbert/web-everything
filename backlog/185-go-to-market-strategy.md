# Go-to-market / marketing strategy — dev-first, show-don't-tell

The launch motion for whichever product becomes the MVP. **Dev-first:** the
*propose-and-verify* demo is the hook (show, don't tell); conformance-as-credibility
is the differentiator.

Scope (a plan, not a build): distribution channels (npm/`npx` primary, GitHub
releases, a VS Code command later — the repo is already a VS Code extension
context); launch surfaces (Show HN / dev communities, a demo video, the
landing+pricing page #184); a **free → paid** funnel (free self-run limited; paid
for scale / CI / team); and a **support model that protects solo bandwidth**
(docs-first deflection, async/community, explicit no-SLA). Stays product-agnostic;
the specific demo swaps per the emerging MVP ([#097](/backlog/097-roadmap-to-mvp/)).
