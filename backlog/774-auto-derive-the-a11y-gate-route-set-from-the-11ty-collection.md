# Auto-derive the a11y gate route set from the 11ty collection/sitemap

**Grounding (✓ ready to ratify):** [#770](/backlog/770-rendered-site-a11y-gate-axe-lane-route-allowlist-we-docs/) shipped a **hand-maintained** route allowlist — [`we:tests/a11y/route-allowlist.ts`](../tests/a11y/route-allowlist.ts) lists 10 stable WE-docs catalog index surfaces; a new published surface is gated only when someone edits that file. This item replaces the hand-list with **auto-derivation** so every published page is gated without a hand-edit. [#763](/backlog/763-gate-ui-best-practices-on-website-changes-rendered-site-a11y/) deferred this as its Fork-3 alternative; [#793](/backlog/793-remediate-we-docs-a11y-violations-flip-the-rendered-site-gat/) sharpened it by flipping 9/10 routes to build-blocking `enforce: true`. Surveyed the auto-derivation prior art (pa11y-ci `--sitemap`, Lighthouse CI `staticDistDir`, large-site a11y sampling) and published [/research/a11y-gate-route-auto-derivation/](/research/a11y-gate-route-auto-derivation/) (report: [`we:reports/2026-06-16-a11y-gate-route-auto-derivation.md`](../reports/2026-06-16-a11y-gate-route-auto-derivation.md)). **The survey reshaped the forks:** the sampling literature added a third scope (index + per-template sample) that beats both poles, and elevated the sitemap mechanism to the prior-art-blessed default — so the two coupled questions each carry a researched default below.

## The axis

Two orthogonal, coupled questions over the existing gate substrate. The gate is a **Playwright lane** ([`we:tests/a11y/rendered-site-a11y.spec.ts`](../tests/a11y/rendered-site-a11y.spec.ts)) that loops `GATED_ROUTES`, runs `AxeBuilder(...).withTags(WCAG_TAGS).analyze()` per route against the 11ty docs origin (`baseURL: http://localhost:8080`), reusing the running dev server.

- **Pages come from pagination over `_data` collections — one template per collection.** Each detail page is generated by a single `*-pages.njk` template: [`we:src/intent-pages.njk:6`](../src/intent-pages.njk#L6) `permalink: "intents/{{ intent.id }}/"`, [`we:src/block-pages.njk:6`](../src/block-pages.njk#L6) `blocks/{{ block.id }}/`, [`we:src/backlog-pages.njk:6`](../src/backlog-pages.njk#L6), [`we:src/research-topic-pages.njk:6`](../src/research-topic-pages.njk#L6) — **26 paginating/permalinked templates**. So a detail-page violation is almost always a **shared-template** bug — one sampled page per template catches it.
- **Index surfaces are the standalone (non-paginating) templates:** `we:src/intents.njk` → `/intents/`, `we:src/blocks.njk` → `/blocks/`, … — the class the current 10-route hand-list holds.
- **A full build emits 2,129 `we:index.html` files** (~790 under `/backlog/{n}/`) — `dir.output: "_site"` ([`we:.eleventy.js:207`](../.eleventy.js#L207)). Scope-A runs axe over all of them.
- **`eleventy --serve` serves from memory — it does *not* populate `_site`** (`dev:docs = eleventy --serve`; only `build:docs = eleventy` writes disk). The Playwright lane boots the dev server, so a `_site/**` glob needs an extra full build the lane doesn't run.
- **No sitemap exists** (`/sitemap.xml` 404s; no `src/*sitemap*`). Authoring one is net-new and independently useful (SEO).
- **The enforce posture is earned, per-route:** #793 flipped 9 routes to `enforce: true`; `/backlog/` is warn-only pending [#805](/backlog/805-flip-backlog-rendered-site-a11y-route-to-enforce-once-backl/). Auto-derivation must preserve this.

### Recommended path at a glance

| Fork | Recommended default | Main alternative | Confidence |
| --- | --- | --- | --- |
| 1 — route-set scope | **Index surfaces + one representative detail page per collection-template** | all-pages · index-only | Med-high |
| 2 — derivation mechanism | **Author `we:sitemap.njk` over `collections.all`; the gate parses it** | glob `_site/**` · reconstruct from `_data` | High |

**Forced invariant (not a fork) — preserve #793's earned enforce posture.** Auto-derivation must keep the 9 routes #793 flipped to `enforce: true` build-blocking. The shape: an **explicit `ENFORCED_ROUTES` set** (data, seeded with the 9), decoupled from the derived set; newly auto-derived routes enter **warn-only** (most-permissive default), promoted per-route as each goes green. The alternative — "let derivation reset every route to warn-only" — is *broken*: it silently un-earns #793 and reintroduces the silent-regression failure mode on 9 routes that were build-blocking. Broken alternative ⇒ this is an invariant, not a `## Fork` (fork-existence test). Classification by the 7-question pass: a **devtools quality gate** (kin to `check:standards`), zero consumer lock-in, **no protocol minted** (a sitemap is the sitemaps.org artifact; axe *is* the WCAG engine — both consumed, not re-specified); mirrored **per-repo** (FUI [#771](/backlog/771-mirror-the-rendered-site-a11y-gate-into-frontier-ui-3001/) derives from its own sitemap). `graduatedTo: none` (a test + a sitemap template + a derivation helper).

## Fork 1 — route-set scope

**Crux:** what does the item's "every published page" resolve to — and how is the CI cost / churn-noise of detail pages handled? The a11y sampling literature ([Pope Tech](https://blog.pope.tech/2024/02/26/improve-effectiveness-and-accuracy-of-web-accessibility-testing-with-sampling/), [TestParty](https://testparty.ai/blog/automated-accessibility-testing-guide)) reframes this: it recommends *against* both poles — "don't test every page on a large site" **and** "homepage-only misses issues" — pointing instead at "representative samples across all page types and **templates**."

- **A — literally every published route** (all `/blocks/{id}/`, `/intents/{id}/`, ~790 `/backlog/{n}/`, …). Matches the item's wording and is maximally self-maintaining, but runs axe over 2,129 pages every CI run (minutes of wall-clock) and gates churn-y in-progress detail pages — the exact cost #763's body flagged, and the extreme the sampling literature discourages.
- **B — auto-discover the catalog *index* surfaces only** (the class the current 10-route list holds), derived from the 11ty collections so a *new* index surface is gated without a hand-edit, but detail pages stay out. Fast and stable, but it is the "homepage-only" anti-pattern — no detail page is ever gated, so a template-introduced violation on `/intents/{id}/` ships silently.
- **C — index surfaces + one representative detail page per collection-template.** Gate every index surface **plus** one sampled detail page per `*-pages.njk` template (~26 extra page loads). Because detail pages share a template, one sample catches the dominant **template-level** regression class — ~95% of A's value at ~1% of its cost — and it stays self-maintaining as new collections appear. The residual gap (a content-specific violation in one item's markdown) is rare and lower-value to gate.

**Default: C — index surfaces + one representative detail page per collection-template.** The prior-art-blessed middle: it removes the hand-maintenance seam #774 targets, gives real detail-page coverage that B lacks, and avoids A's all-pages CI blow-up — while the one-template-per-collection tree structure makes a single sample genuinely representative. (Red-team note for the deciding turn: C's blind spot is a per-item *content* violation that the sampled page happens not to hit; acceptable because such violations are rare, low-leverage, and the warn-only ratchet still surfaces them if the sample rotates — but worth a skeptic pass on whether `/backlog/` content specifically warrants a larger sample given its ~790 author-written bodies.)

## Fork 2 — derivation mechanism

**Crux:** there is no sitemap today, so this decides the *source of truth* the Playwright gate reads to get its URL list. Each mechanism couples the test to a different artifact.

- **A — author a `we:sitemap.njk` over `collections.all`; the gate fetches and parses it.** An 11ty template emits `/sitemap.xml` from 11ty's own complete page knowledge; the test reads it over HTTP (the same origin it already hits). This is the canonical auto-derivation — pa11y-ci's [`--sitemap`](https://github.com/pa11y/pa11y-ci/blob/main/README.md) consumes exactly a sitemap.xml — and it doubles as the missing SEO sitemap. Scope-C composes cleanly: emit all pages, the gate keeps every index surface + the first detail page per path-prefix group.
- **B — glob the built `_site/**/index.html`.** *Rejected* — `eleventy --serve` serves from memory and doesn't write `_site`, so the dev-server-reusing lane would need an extra full `build:docs` first; and Lighthouse CI's `staticDistDir` shows the nested-dir glob is fiddly ([issue #95](https://github.com/GoogleChrome/lighthouse-ci/issues/95): only root-level files collected) — exactly our deeply-nested `/intents/{id}/` output.
- **C — reconstruct permalinks from the `_data` collections.** *Rejected* — the test re-implements the permalink patterns that already live in 26 `*-pages.njk` templates; the two drift the moment a permalink changes (the gate would gate a stale or wrong URL silently).

**Default: A — author `we:sitemap.njk` over `collections.all`, parse it in the gate.** The prior-art-blessed, 11ty-native path: one source of truth (collections.all), the test consumes a published standard artifact rather than reaching into build internals or re-deriving routes, and it ships a real SEO sitemap as a side benefit. Decouples the gate from both the build-output layout (B) and the template permalink logic (C), per the separation bias.

## Ruling (ratified 2026-06-17)

**Fork 1 → C** (index surfaces + one representative detail page per collection-template), ~80%.
Both poles are disqualified on merit: A (all 2,129 pages) runs axe over ~790 churn-y backlog bodies
and is the sampling-literature anti-pattern; B (index-only) is the homepage-only anti-pattern that
leaves every detail template ungated — the exact gap #774 closes. C catches the dominant
template-level regression class (one template ⇒ one sample is representative) at ~1% of A's cost.
*Residual:* `/backlog/`'s ~790 hand-authored bodies can harbor per-item content violations a single
sample misses (missing alt, skipped heading) — rare, surfaced by the warn-only ratchet; revisit a
larger backlog sample only if one actually appears, not pre-emptively.

**Fork 2 → A** (author `we:sitemap.njk` over `collections.all`; the gate fetches + parses it), ~90%.
B (glob `_site/**`) is rejected — `eleventy --serve` serves from memory and never writes `_site`, so
the dev-server-reusing lane would need an extra full `build:docs`. C (reconstruct permalinks from
`_data`) is rejected — re-implements the 26 `*-pages.njk` permalink patterns and drifts silently. A
reads the sitemap over the same HTTP origin the lane already hits (zero extra build), consumes a
published standard artifact, and ships the missing SEO `/sitemap.xml`. Scope-C composes as a filter
over it (keep every index + first detail page per path-prefix group).

**Forced invariant** stands: newly-derived routes enter warn-only; an explicit `ENFORCED_ROUTES`
set seeded with #793's 9 stays build-blocking. #805 now resolved ⇒ that set seeds at the final 10/10
posture. `graduatedTo: none`.
