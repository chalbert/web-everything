One challenge I regularly had while buidling a component library was that designers often came with visual variation of the same core components. Let's take a navigation, like the one we worked one. Many variation can be done on the same subject. Some or structural and probably warrant a completely new block/component, let's say an accordeon navigation with collapses. But sometimes it's mostly visual, but potencially styles that cannot purely be achieved though styles. Sometimes icons have to be injected, or maybe a status or notification number, or pop over, or anything else. 

In such a case, we have a perfectly good navigation component that answer to all accessiblity requirement, and it would be a waste not to use it. With React, which is very good for composition, we have quite a few strategies. We can make the navigation take a CustomLink component to completely replace a sub component. We can pass a function to decoration, HOC style, the existing one. We can pass props to be merged into the existing one. We can create a branch new component but reusing part of the base navigation that would be split into abstract piece. The same component can even bu seamlessly configured using a context, and the same component can behave very differently on 2 differents app.

I would like to explore the strategies that could be use in the context of a HTML-first, standard based approach using WE components.
